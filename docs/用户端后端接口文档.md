# 用户端后端接口文档（MVP）

## 1. 基本信息

- 服务目录：`services/user-backend`
- 默认端口：`3001`
- 基础地址：`http://localhost:3001`
- 数据格式：`application/json`

## 2. 鉴权说明

当前用户端后端仍使用临时请求头 `x-user-id` 作为用户身份标识，后续会替换为真实微信登录态。

- 需要鉴权的接口：`/api/addresses/*`、`/api/cart/*`、`/api/orders/*`、`/api/payments/prepay`
- `POST /api/payments/callback` 是模拟外部支付回调，不需要 `x-user-id`

请求头示例：

```http
x-user-id: 1
Content-Type: application/json
```

缺少或非法时返回：

```json
{
  "message": "缺少有效的 x-user-id"
}
```

## 3. 认证与配置

### POST /api/auth/mock-login

- 说明：模拟登录，按 `mockOpenId` 创建或复用用户
- 请求头：无要求
- 请求体：

```json
{
  "mockOpenId": "codex_test_user_001",
  "nickname": "测试用户",
  "avatarUrl": "https://example.com/avatar.png",
  "unionid": "union_001",
  "phone": "13800000000"
}
```

- 返回示例：

```json
{
  "userId": 1,
  "profile": {
    "openid": "codex_test_user_001",
    "unionid": "union_001",
    "nickname": "测试用户",
    "avatarUrl": "https://example.com/avatar.png",
    "phone": "13800000000",
    "status": "active"
  }
}
```

### GET /api/app-config

- 说明：获取前端运行所需的应用配置
- 请求头：无要求
- 返回示例：

```json
{
  "wristMinCm": 13,
  "wristMaxCm": 21,
  "shippingFee": 12,
  "paymentChannel": "wechat"
}
```

## 4. 地址接口

### GET /api/addresses

- 说明：获取当前用户地址列表，默认地址优先
- 请求头：需要 `x-user-id`

### POST /api/addresses

- 说明：新增地址
- 请求体字段：`receiverName`、`receiverPhone`、`province`、`city`、`district`、`detailAddress`、`postalCode`、`tag`、`isDefault`
- 请求头：需要 `x-user-id`

### PUT /api/addresses/:id

- 说明：更新地址
- 请求头：需要 `x-user-id`

### PATCH /api/addresses/:id/default

- 说明：设置默认地址，会清空当前默认地址
- 请求头：需要 `x-user-id`

### DELETE /api/addresses/:id

- 说明：删除地址
- 请求头：需要 `x-user-id`

## 7. 购物车接口

### GET /api/cart

- 说明：获取当前用户购物车，不存在时自动创建
- 请求头：需要 `x-user-id`
- 返回示例：

```json
{
  "cartId": 1,
  "items": [
    {
      "id": 10,
      "bead_id": 1,
      "quantity": 2,
      "selected": 1,
      "name": "白水晶6mm",
      "color": "白水晶",
      "size_mm": "6.0",
      "unit_price": "3.00"
    }
  ]
}
```

### POST /api/cart/items

- 说明：把珠子加入购物车，已存在同珠子会累加数量
- 请求头：需要 `x-user-id`
- 请求体：

```json
{
  "beadId": 1,
  "quantity": 2,
  "wristSizeCm": 16
}
```

- 规则：
  - `beadId` 必填，且必须是正整数
  - `quantity` 可选，默认 `1`
  - `wristSizeCm` 必填，必须在 `13-21`
  - 购物车内所有已选中珠子的累计直径不得超过 `wristSizeCm * 10`

### PATCH /api/cart/items/:id

- 说明：更新购物车项
- 请求头：需要 `x-user-id`
- 请求体支持字段：`quantity`、`selected`、`wristSizeCm`

```json
{
  "quantity": 3,
  "selected": 1,
  "wristSizeCm": 16
}
```

- 规则：
  - `quantity` 必须是正整数
  - `selected` 必须是 `0/1`
  - 至少提供 `quantity` 或 `selected` 之一
  - 更新后如果结果仍为选中状态，则必须提供 `wristSizeCm`
  - 当结果为选中状态时，仍会重新校验珠子累计直径上限

### DELETE /api/cart/items/:id

- 说明：删除购物车项
- 请求头：需要 `x-user-id`

## 8. 订单接口

### POST /api/orders

- 说明：基于购物车已选中商品创建订单
- 请求头：需要 `x-user-id`
- 请求体：

```json
{
  "wristSizeCm": 16,
  "addressId": 1,
  "remark": "尽快发货"
}
```

- 规则：
  - `wristSizeCm` 必须在 `13-21`
  - `addressId` 必须属于当前用户
  - 购物车必须存在 `selected=1` 的商品
  - 下单时会再次校验珠子累计直径上限

### GET /api/orders

- 说明：获取当前用户订单列表，倒序排列
- 请求头：需要 `x-user-id`

### GET /api/orders/:id

- 说明：获取订单详情
- 请求头：需要 `x-user-id`
- 返回包含：订单主信息、订单明细、地址快照、物流摘要、退款摘要

### POST /api/orders/:id/cancel

- 说明：取消订单
- 请求头：需要 `x-user-id`
- 规则：仅 `pending_payment` 订单可取消

### POST /api/orders/:id/refund

- 说明：发起退款申请
- 请求头：需要 `x-user-id`
- 请求体：

```json
{
  "reason": "商品不适合"
}
```

- 规则：
  - 仅已支付订单可发起退款
  - 当前已有退款流程中的订单不能重复申请
  - 成功后订单进入 `refund_pending`

## 9. 支付接口

### POST /api/payments/prepay

- 说明：创建预支付单并返回模拟支付参数
- 请求头：需要 `x-user-id`
- 请求体：

```json
{
  "orderId": 1001
}
```

- 规则：
  - 仅 `pending_payment` 订单可发起
  - 若该订单已存在 `initiated` 状态支付单，则直接复用该支付单并返回同一组参数
  - 若不存在，则新建支付单并返回新的 `paymentNo`

### POST /api/payments/callback

- 说明：模拟外部支付平台回调，当前不需要 `x-user-id`
- 请求头：无要求
- 请求体：

```json
{
  "paymentNo": "PAY171254000000012345",
  "success": true,
  "transactionId": "wx_tx_123456"
}
```

- 规则：
  - `paymentNo` 必填
  - `success=true` 时会将订单流转到 `pending_production`
  - 接口支持幂等处理，重复回调不会重复推进状态

## 10. 状态码约定

- `200`：请求成功
- `201`：创建成功
- `400`：参数错误或业务校验失败
- `401`：缺少或非法身份标识
- `404`：资源不存在
- `500`：服务内部错误

## 11. 当前版本限制

1. 用户端鉴权仍是临时方案 `x-user-id`，尚未接入真实微信登录态。
2. 支付仍是 mock 回调流程，尚未接入真实微信支付网关。
3. 订单、购物车和地址接口均已按当前实现收口，前端联调时应以本文件为准。
