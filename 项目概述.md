# 水晶手链定制小程序 - 项目概述（v1.1）

## 1. 产品定位

- **产品名称**：水晶手链定制微信小程序
- **核心功能**：用户选择不同颜色、大小的水晶珠子，搭配组合成自己喜欢的手链；支持输入腕围，并在珠子数量达到极限时禁止下单
- **商业模式（当前）**：按每颗珠子定价
- **商业模式（二期预留）**：预设套餐一键下单（较自由搭配有优惠）
- **合作分工**：程序部分独立开发，资质/供应链/物流/客服/运营由合作方负责

---

## 2. 业务流程

### 2.1 核心链路（用户端）

```text
微信授权登录 → 输入腕围（13-21cm）→ 选择珠子（颜色+尺寸）→ 搭配组合
→ 系统判断珠子数量是否超限（超限禁止下单）→ 加入购物车
→ 提交订单 → 支付 → 待制作 → 待发货 → 已完成
```

### 2.2 发货链路（后台发货端）

```text
后台查看待发货订单 → 录入物流公司/运单号
→ 用户端可查看物流信息 → 订单完成
```

### 2.3 已确认业务规则

- [x] 定价策略：按每颗珠子定价
- [x] SKU 维度（首版）：颜色 + 尺寸
- [x] SKU 扩展（二期预留）：材质、净度、产地
- [x] 库存策略（首版）：无限库存
- [x] 腕围输入范围：13-21cm，超出范围不可购买
- [x] 珠子上限规则：珠子按颗添加，累计直径总和超过腕围时不可继续添加
- [x] 超限处理：禁止下单
- [x] 必要功能：购物车
- [x] 登录方式：微信授权登录
- [x] 地址功能：支持多地址、历史地址编辑、默认地址
- [x] 订单状态扩展：支持已取消、退款中、已退款
- [x] 运费策略：固定运费
- [x] 发货能力：发货端后台手动填写物流公司与物流单号
- [x] 首版上线日期：2026-04-08

## 6. 数据库设计清单（按角色划分）

### 6.1 用户端核心表

- `users`：用户信息（微信标识、昵称等）
- `beads`：珠子基础信息（颜色、尺寸、单价）
- `cart`：购物车主表
- `cart_items`：购物车明细
- `addresses`：收货地址（含默认地址）
- `orders`：订单主表（金额、运费、状态）
- `order_items`：订单明细（每颗珠子的快照信息）
- `payments`：支付记录与回调记录

### 6.2 后台发货端核心表

- `shipments`：发货信息（物流公司、运单号、发货时间、发货备注）
- `order_status_logs`：订单状态流转日志（便于追踪退款/取消）
- `refunds`：退款单记录（申请时间、状态、金额、原因）

### 6.3 二期预留表

- `packages`：套餐主表
- `package_items`：套餐明细
- `inventory_logs`：库存变更流水（启用真实库存时使用）
- `coupon_templates` / `user_coupons`：优惠券体系（启用营销功能时使用）
