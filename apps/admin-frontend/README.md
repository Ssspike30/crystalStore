# crystalStore 管理台静态联调页

这是一个零依赖的后台前端原型，围绕 `services/admin-backend` 已有接口实现最小运营闭环。

支持的核心动作：

- 配置后台服务根地址和 `x-admin-key`
- 查看订单列表、筛选订单、打开订单详情
- 切换订单 / 退款分页与每页条数
- 对待制作订单录入发货信息
- 对待收货订单手动完结
- 对待付款订单取消
- 查看退款列表并对待审核退款执行通过 / 驳回
- 在后台接口不可达时自动回退到本地 demo 模式

## 启动方式

### 1. 直接打开

直接双击 `index.html` 即可。

### 2. 使用静态服务器

如果你希望通过 `http://` 访问，可以在当前目录执行：

```powershell
python -m http.server 8081
```

然后打开：

```text
http://127.0.0.1:8081/
```

## 真实后台联调

1. 先启动 `services/admin-backend`
2. 在页面顶部填入：
   - 后台服务根地址，例如 `http://localhost:3002`
   - `x-admin-key`，值应等于后端环境变量 `ADMIN_API_KEY`
3. 点击“保存并连接”

如果 `GET /health` 不可达，页面会自动退回 demo 模式。
如果健康检查成功，但 `x-admin-key` 错误，列表请求会直接显示后端错误，不会偷偷切回 demo。
如果你误填成 `http://localhost:3002/api/admin`，页面会自动收口为服务根地址。

## 当前限制

- 这是静态联调页，不是正式后台工程
- 没有后台登录体系，当前仅支持手工输入 `x-admin-key`
- 珠子管理、运费配置等后台功能还未纳入本页
- 真实联调依赖后台允许当前页面来源访问接口
