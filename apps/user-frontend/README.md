# crystalStore 用户端最小闭环联调页

这是一个零依赖的静态页版本，用来跑通用户链路：

`mock 登录 -> 选珠 -> 购物车 -> 地址 -> 下单 -> prepay -> mock callback -> 订单列表/详情 -> 退款申请`

## 启动方式

### 1. 直接打开

直接双击 `index.html` 即可。

这种方式会默认使用本地演示数据，不依赖后端，也不需要安装任何包。

### 2. 使用简易静态服务器

如果你希望通过 `http://` 地址访问页面，可以在当前目录启动一个简单静态服务器。例如 PowerShell 下：

```powershell
python -m http.server 8080
```

然后打开：

```text
http://127.0.0.1:8080/
```

## API_BASE_URL 配置

页面顶部提供了 `API_BASE_URL` 输入框。

可填写例如：

```text
http://localhost:3001
```

说明：

1. 页面会优先尝试真实接口。
2. 如果真实后端不可用、跨域失败，页面会自动回退到本地演示模式。
3. 如果你想稳定连接真实后端，后端需要允许跨域，或者把静态页和后端放到同源环境里。

## 手工测试路径

建议按下面顺序验证：

1. 打开页面后点击 `Mock 登录`。
2. 在 `腕围输入` 填一个 13 到 21 之间的值。
3. 在珠子列表里选择商品并加入购物车。
4. 在购物车里尝试勾选、改数量、删除。
5. 在地址管理里新增地址，或编辑默认地址。
6. 点击 `去下单` 创建订单。
7. 在支付区域点击 `发起 prepay`。
8. 点击 `模拟 callback 成功`，检查订单状态变为已支付/待制作。
9. 在订单列表里点开详情，确认支付记录与状态流转。
10. 对已支付订单点击 `申请退款`，确认退款记录出现。

## 演示数据

页面内置了本地 mock 数据与示例珠子，可以直接用于联调演示。

如果想回到初始状态，点击顶部的 `重置演示数据`。
